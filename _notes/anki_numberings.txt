What does \(\alpha(n) \simeq \beta(n)\) mean for partial functions?	The values \(\alpha(n)\) and \(\beta(n)\) are either <b>both undefined</b> or are <b>both defined and equal</b>.	numberings::basics
What is the principal (universal) function of a sequence \((\alpha_e)_{e \in \mathbb{N}}\)?	The partial function \(\alpha: \mathbb{N}^2 \rightharpoonup \mathbb{N}\) defined by \(\alpha(e, x) \simeq \alpha_e(x)\) for all \(e, x \in \mathbb{N}\). It lets you treat a sequence of unary functions as a single binary function and vice versa.	numberings::basics
What is a numbering (of partial functions)?	A sequence \((\alpha_e)_{e \in \mathbb{N}}\) of partial functions whose principal function \(\alpha(e, x) \simeq \alpha_e(x)\) is <b>partial computable</b>. This means there is a single algorithm (universal machine) that given \(e\) and \(x\) simulates "program \(e\) on input \(x\)."	numberings::basics
What is the programming-system intuition for a numbering?	The index \(e\) is the "program code," \(\alpha_e\) is the partial function computed by program \(e\), and the principal function \(\alpha(e,x)\) is the universal interpreter: "run program \(e\) on input \(x\)." Partial computability of \(\alpha\) means this interpreter exists.	numberings::basics
When is a numbering \((\alpha_e)\) universal for a class \(\mathcal{F}\)?	When \(\mathcal{F} = \{\alpha_e : e \in \mathbb{N}\}\). A <b>universal numbering</b> specifically enumerates <b>all</b> partial computable functions.	numberings::basics
Can a numbering have duplicate indices (different \(e\) values giving the same function)?	Yes. No injectivity requirement exists. Different programs can compute the same function (dead code, reordering, etc.). For <b>acceptable numberings</b>, duplicates are unavoidable: by the padding lemma, every function has <b>infinitely many</b> indices.	numberings::basics
In a numbering \((\alpha_e)\), are all \(\alpha_e\) partial computable?	Yes. Given a TM computing the principal function \(\alpha(e,x)\), modify it to first transform input \(x\) into \(\langle e, x \rangle\), then compute. This yields a TM for \(\alpha_e\). Consequently, a numbering is universal iff all partial computable functions occur among the \(\alpha_e\).	numberings::basics
Can every sequence of computable functions form a numbering?	No. The principal function must itself be partial computable. Counterexample: fix an undecidable set \(A\), define \(\alpha_e\) as the constant function \(1\) if \(e \in A\), else constant \(0\). Then \(e \mapsto \alpha(e,e)\) would be the characteristic function of \(A\), contradicting undecidability.	numberings::basics
State the Universal Turing Machine theorem.	There exists a universal numbering — a numbering \((\phi_e)_{e \in \mathbb{N}}\) that enumerates all partial computable functions.	numberings::universal_TM
Outline the proof of the Universal TM theorem (Step 1: enumerate TMs).	Fix a syntax for TMs. Enumerate all finite strings \(w_0, w_1, \dots\) in length-lexicographic order. If \(w_e\) is a valid TM code, let \(M_e\) be that machine; otherwise let \(M_e\) be a dummy machine that never halts. Every TM appears as some \(M_e\).	numberings::universal_TM
Outline the proof of the Universal TM theorem (Step 2–3: principal function is partial computable).	Define \(\phi(e,x) \simeq\) "output of \(M_e\) on \(x\)." A universal simulator \(U\) on input \(\langle e,x \rangle\): (1) computes \(w_e\), (2) checks if it's a valid TM code (if not, loops), (3) simulates \(M_e\) on \(x\) step-by-step, (4) outputs result if it halts. This is a TM, so \(\phi\) is partial computable.	numberings::universal_TM
Outline the proof of the Universal TM theorem (Step 4: universality).	Any partial computable \(f\) is computed by some TM \(M\). Since every TM appears in the list, pick \(e\) with \(M_e = M\). Then \(\phi_e = f\). So \(\{\phi_e : e \in \mathbb{N}\}\) = all partial computable functions.	numberings::universal_TM
What is the standard enumeration of Turing machines?	The sequence \(M_0, M_1, M_2, \dots\) constructed in the Universal TM theorem proof: enumerate all strings in length-lexicographic order, assign valid codes to their TMs and invalid codes to a dummy never-halting machine.	numberings::universal_TM
What is the standard enumeration of partial computable functions?	The sequence \(\phi_0, \phi_1, \phi_2, \dots\) where \(\phi_i\) is the partial function computed by \(M_i\) from the standard enumeration of TMs. This is a universal numbering.	numberings::universal_TM
State the theorem: there is no numbering of all total computable functions.	There exists no numbering whose range is exactly the set of all (total) computable functions.	numberings::no_total_numbering
Prove that no numbering of all total computable functions exists.	Suppose \((\alpha_e)\) is such a numbering. Since every \(\alpha_e\) is total, the principal function \(\alpha(e,x) = \alpha_e(x)\) is total and computable. Define \(d(e) = \alpha(e,e) + 1\), which is total computable. But \(d(e) = \alpha_e(e) + 1 \neq \alpha_e(e)\), so \(d \neq \alpha_e\) for all \(e\) — contradiction.	numberings::no_total_numbering
Why doesn't the same diagonalization argument disprove the existence of a universal numbering?	In a universal numbering, \(\alpha(e,e)\) may be <b>undefined</b>. The diagonal \(d(e) \simeq 1 + \alpha(e,e)\) is then <b>partial</b>. When \(\alpha_e(e) \uparrow\), also \(d(e) \uparrow\), so there's no disagreement at \(e\). If \(d = \alpha_k\), then \(\alpha(k,k) \simeq 1 + \alpha(k,k)\) forces \(\alpha(k,k) \uparrow\) — no contradiction.	numberings::no_total_numbering
Why can't you "fix" the diagonal to make it total and disprove the universal numbering?	You might try \(g(e) = 1 + \alpha(e,e)\) if defined, else \(0\). This would be total and diagonalize, but computing it requires deciding whether \(\alpha(e,e)\) halts — i.e., solving the halting problem. So \(g\) is <b>not computable</b>.	numberings::no_total_numbering
What is the key property that makes diagonalization work against total functions but not partial ones?	<b>Totality of the principal function.</b> When all \(\alpha_e\) are total, \(\alpha(e,e)\) is always defined, so \(d(e) = \alpha(e,e)+1\) is total and forces disagreement everywhere. When functions can be partial, the diagonal can be undefined exactly where needed, avoiding contradiction.	numberings::no_total_numbering
What is the diagonal halting set \(H_\alpha\) for a numbering \((\alpha_e)\)?	\(H_\alpha = \{e \in \mathbb{N} : \alpha_e(e) \downarrow\}\) — the set of indices \(e\) for which program \(e\) halts on its own code as input.	numberings::halting
State the lemma about the diagonal halting set.	For any numbering \((\alpha_e)\): (1) \(H_\alpha\) is r.e. (2) If the numbering contains every total computable function, then \(H_\alpha\) is <b>not decidable</b>.	numberings::halting
Prove that \(H_\alpha\) is r.e.	The function \(d(e) \simeq \alpha(e,e)\) is partial computable (since \(\alpha\) is). Its domain is \(\{e : \alpha(e,e) \downarrow\} = H_\alpha\). A set is r.e. iff it's the domain of a partial computable function, so \(H_\alpha\) is r.e.	numberings::halting
Prove that if the numbering contains all total computable functions, then \(H_\alpha\) is undecidable.	By contrapositive: assume \(H_\alpha\) is decidable. Define \(g(e) = 1 + \alpha_e(e)\) if \(e \in H_\alpha\), else \(0\). Then \(g\) is total computable. But \(g(e) \neq \alpha_e(e)\) for all \(e\) (either \(g(e) = \alpha_e(e)+1\) or \(g(e)=0\) while \(\alpha_e(e) \uparrow\)). So \(g\) is a total computable function not in the numbering — contradicting that it contains all total computable functions.	numberings::halting
Define the diagonal halting problem \(H\) and the general halting problem \(H_{\text{gen}}\).	For the standard numbering \((\phi_e)\):<br>\(H = H_{\text{diag}} = \{e \in \mathbb{N} : \phi_e(e) \downarrow\}\)<br>\(H_{\text{gen}} = \{\langle e, x \rangle \in \mathbb{N} : \phi_e(x) \downarrow\}\)<br>\(H\) asks "does program \(e\) halt on itself?" and \(H_{\text{gen}}\) asks "does program \(e\) halt on input \(x\)?"	numberings::halting
State the theorem about the decidability status of the halting problem \(H\).	\(H\) is recursively enumerable but <b>not decidable</b>. Its complement \(\mathbb{N} \setminus H\) is <b>not r.e.</b>	numberings::halting
Prove the halting problem theorem.	The standard numbering is universal, so it contains all computable functions. By the diagonal halting set lemma, \(H\) is r.e. but not decidable. If \(\mathbb{N} \setminus H\) were also r.e., then by the complement characterization (\(A\) decidable \(\iff\) \(A\) and \(\bar{A}\) both r.e.), \(H\) would be decidable — contradiction. So \(\mathbb{N} \setminus H\) is not r.e.	numberings::halting
What is a Gödel numbering (acceptable numbering)?	A numbering \((\beta_e)\) such that for <b>every</b> numbering \((\alpha_e)\), there exists a <b>computable</b> function \(f\) with \(\alpha_e = \beta_{f(e)}\) for all \(e\). Equivalently: every other numbering can be effectively translated (compiled) into \((\beta_e)\).	numberings::godel
What is the role of the function \(f\) in the definition of a Gödel numbering?	\(f\) acts as a <b>compiler/translator</b> from indices in the \(\alpha\)-numbering to equivalent indices in the \(\beta\)-numbering: program \(e\) in system \(\alpha\) computes the same function as program \(f(e)\) in system \(\beta\).	numberings::godel
What is an admissible numbering? Give the two conditions.	A numbering \((\alpha_e)\) of partial computable functions is admissible if:<br>(1) <b>Universality</b>: it enumerates all partial computable functions.<br>(2) <b>s-m-n property</b>: there is a total computable \(s(e,x)\) such that \(\alpha_{s(e,x)}(y) \simeq \alpha_e(\langle x, y \rangle)\). Intuition: from an index for a 2-argument program and a value \(x\), you can compute an index for the 1-argument program with \(x\) hardwired.	numberings::godel
What is the equivalent characterization of admissible numberings (Rogers' translation property)?	A universal numbering \(\alpha\) is admissible iff for every other numbering \(\beta\), there exists a total computable translator \(t\) with \(\beta_e = \alpha_{t(e)}\) for all \(e\). This is exactly the Gödel numbering definition.	numberings::godel
Why does admissibility matter beyond mere universality?	A universal numbering lists all partial computable functions, but might be "unnatural" — basic operations like "fix the first input" might not be computable at the level of indices. Admissibility guarantees the numbering behaves like a reasonable programming system where you can compile and partial-evaluate effectively.	numberings::godel
What is a classic Gödel numbering in mathematical logic?	An encoding assigning each symbol a unique number, then encoding a formula \(s_1 \dots s_k\) as \(G(s_1 \dots s_k) = 2^{c(s_1)} \cdot 3^{c(s_2)} \cdot 5^{c(s_3)} \cdots p_k^{c(s_k)}\) using prime powers. Unique prime factorization makes this reversible. Developed by Gödel for his incompleteness theorems.	numberings::godel
Prove that the standard numbering \((\phi_e)\) is a Gödel numbering.	Let \((\alpha_e)\) be any numbering with principal function computed by TM \(M\). For any \(e\), construct \(M'\) that on input \(x\) first forms \(\langle e, x \rangle\) then simulates \(M\). This \(M'\) computes \(\alpha_e\). The construction of \(M'\) from \(e\) is effective, so we can computably find its index \(f(e)\) in the standard enumeration. Thus \(\phi_{f(e)} = \alpha_e\) for all \(e\), with \(f\) computable.	numberings::godel
Is every universal numbering a Gödel numbering?	No. There exist universal numberings that are not Gödel numberings — they enumerate all partial computable functions but lack the translation property (cannot effectively compile from every other numbering).	numberings::godel
What is Rogers' equivalence theorem (informally)?	All acceptable (Gödel/admissible) numberings are equivalent to each other — there exist computable translations in both directions between any two of them. They all represent equally powerful programming systems.	numberings::godel
Summarize the hierarchy: numbering → universal numbering → Gödel numbering.	<b>Numbering</b>: sequence with partial computable principal function (some partial computable functions listed).<br><b>Universal numbering</b>: numbering that lists <b>all</b> partial computable functions.<br><b>Gödel numbering</b>: universal numbering where every other numbering can be effectively translated into it (has s-m-n property). Every Gödel ⇒ universal ⇒ numbering, but not vice versa.	numberings::godel
