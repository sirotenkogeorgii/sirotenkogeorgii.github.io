What is the purpose of generalized autoregressive models?	They extend the autoregressive framework to model <b>non-Gaussian</b> time series (binary sequences, count data) by combining AR structure with <b>Generalized Linear Models (GLMs)</b>.	mbtsa::generalized_ar::motivation
Describe the AR model for binary processes.	<b>Data:</b> binary time series \(X_t \in \{0, 1\}\).<br><b>Model:</b> \(X_t \mid X_{t-1}, \ldots, X_{t-p} \sim \mathrm{Bernoulli}(\pi_t)\)<br><b>Linear predictor:</b> \(\eta_t = c + \sum_{i=1}^p \alpha_i X_{t-i}\)<br><b>Link function (sigmoid):</b> \(\pi_t = \frac{1}{1 + e^{-\eta_t}}\)<br><br>This is logistic regression where the predictors are <b>lagged values</b> of the series itself. No closed-form solution — requires numerical optimization (GD or Newton-Raphson).	mbtsa::generalized_ar::binary_ar
What are the main limitations of binary AR models?	(1) <b>Unstable estimates:</b> for strongly dependent series, certain histories (e.g., long runs of 1s) can lead to perfect prediction, causing MLE coefficients to become very large or non-existent.<br>(2) <b>Limited dynamical structure:</b> struggle to capture complex patterns like bursts of activity vs. silence.<br>(3) <b>Scalability:</b> extension to the multivariate case is challenging.<br>(4) <b>Numerical inference:</b> parameter inference is purely numerical.<br><br>Common remedy: reduce model order \(p\) to reduce complexity.	mbtsa::generalized_ar::binary_ar
Define the change point model for binary time series and its parameters.	Models the success probability \(\pi_t\) as a sigmoid function of time:<br><br>\(\pi_t = m + \frac{d}{1 + e^{-a(t - c)}}\)<br><br>Parameters \(\theta = \{m, d, a, c\}\):<br>- \(m\): baseline probability before the change.<br>- \(d\): amplitude of the change (probability after = \(m + d\)).<br>- \(a\): inverse slope (\(a \ge 0\)); smaller \(a\) = steeper, more abrupt change.<br>- \(c\): change point location in time.<br><br><b>Limitation:</b> only allows a single, monotonic change point.	mbtsa::generalized_ar::change_point
How are parameters estimated in the change point model?	<b>Likelihood:</b> \(P(X \mid \theta) = \prod_{t=1}^T \pi_t^{X_t}(1-\pi_t)^{1-X_t}\)<br><br><b>Log-likelihood:</b> \(\ell(\theta) = \sum_{t=1}^T [X_t \log(\pi_t) + (1-X_t)\log(1-\pi_t)]\)<br><br>where \(\pi_t = m + \frac{d}{1+e^{-a(t-c)}}\). Parameters are found via <b>MLE</b> using numerical optimization.	mbtsa::generalized_ar::change_point
Describe the Poisson GLM for count processes (AR model for counts).	<b>Data:</b> count process \(C_t = (C_{1t}, \ldots, C_{Nt})^\top\), \(C_{it} \in \{0,1,2,\ldots\}\).<br><b>Model:</b> \(C_{it} \mid \text{past} \sim \mathrm{Poisson}(\lambda_{it})\)<br><br>Key Poisson property: \(\mathbb{E}[C_{it} \mid \text{past}] = \mathrm{Var}(C_{it} \mid \text{past}) = \lambda_{it}\)<br><br><b>Rate model (AR with exp link):</b><br>\(\lambda_t = \exp\!\left(c + \sum_{j=1}^p A_j C_{t-j}\right)\)<br><br>The \(\exp\) ensures positivity of rates. Matrices \(A_j\) capture "effective couplings" between processes.	mbtsa::generalized_ar::poisson_glm
Write the log-likelihood for the Poisson GLM and state its optimization property.	\(\ell(\{c, A_j\}) = \sum_{t,i} \left[C_{it}\!\left(c_i + \sum_{j,k}(A_j)_{ik} C_{k,t-j}\right) - \exp\!\left(c_i + \sum_{j,k}(A_j)_{ik} C_{k,t-j}\right)\right] + \text{const}\)<br><br>No closed-form solution, <b>but</b> the Poisson log-likelihood is <b>concave</b>. This guarantees that numerical optimization (e.g., gradient descent) converges to the <b>unique global maximum</b>.	mbtsa::generalized_ar::poisson_glm
