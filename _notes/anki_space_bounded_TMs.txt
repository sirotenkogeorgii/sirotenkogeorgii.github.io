What is an off-line Turing machine?	A TM with a separate <b>input tape</b> (read-only, head can only access input symbols and two adjacent blanks) and <b>output tape</b> (head can only move right; must advance after each write). A \(k\)-tape off-line TM has \(k\) work tapes in addition to these.	space::off_line_TM
How are configurations defined for off-line TMs?	Like standard TMs (state, work tape contents, work tape head positions) but: (1) <b>Output tape</b> content and head position are excluded (they can't influence future steps). (2) <b>Input tape</b>: only the head position is included (an integer in \(\{0, \dots, n+1\}\)), not the content (which is read-only and fixed).	space::off_line_TM
Why use off-line TMs for space-bounded computation?	For sublinear space bounds (e.g., \(\log n\)), we must separate the memory used for input/output from computation memory. Off-line TMs let configurations be smaller than the input itself, which is essential for meaningful sublinear space classes.	space::off_line_TM
What is the space usage \(\text{space}_M(w)\) of a deterministic TM \(M\) on input \(w\)?	The maximum number of cells accessed on a single work tape during the computation, if \(M\) terminates on \(w\). Undefined (\(\uparrow\)) if \(M\) does not terminate.	space::deterministic
What is a space bound?	A computable function \(s: \mathbb{N} \to \mathbb{N}\) with \(s(n) \geq \log n\) for all \(n > 0\).	space::deterministic
When is a deterministic TM \(M\) called \(s(n)\)-space-bounded?	When \(M\) is total and for all but finitely many inputs \(w\), \(\text{space}_M(w) \leq s(|w|)\).	space::deterministic
Define \(\text{DSPACE}(s(n))\).	\(\text{DSPACE}(s(n)) = \{L \subseteq \{0,1\}^* : L = L(M) \text{ for a deterministic } s(n)\text{-space-bounded TM } M\}\)	space::deterministic
Define the classes LOG, PSPACE, and EXPSPACE.	\(\text{LOG} = \text{DSPACE}(\log)\), \(\text{PSPACE} = \text{DSPACE}(\text{poly})\), \(\text{EXPSPACE} = \text{DSPACE}(2^{\text{poly}})\). Here \(\log = \{n \mapsto c \log n + c\}\) and \(\text{poly} = \{n \mapsto n^c + c\}\) for \(c \in \mathbb{N} \setminus \{0\}\).	space::deterministic
What is \(\text{FSPACE}(s(n))\)?	The class of functions \(f: \{0,1\}^* \to \{0,1\}^*\) computable by a deterministic \(s(n)\)-space-bounded TM. The output is the word written on the output tape upon termination.	space::deterministic
State the Linear Compression theorem for space.	For every space bound \(s\) and constant \(c\): \(\text{DSPACE}(c \cdot s(n)) \subseteq \text{DSPACE}(s(n))\). Constant factors in space bounds don't matter. Also holds for NSPACE.	space::robustness
State the Alphabet Change theorem.	For any space bound \(s\) and \(k \geq 2\): every language in \(\text{DSPACE}_k(s(n))\) can be recognized by a deterministic \(O(s(n))\)-space-bounded \(k\)-tape TM with tape alphabet \(\{0, 1, \square\}\). The tape alphabet can always be reduced to binary + blank.	space::robustness
When is a nondeterministic TM \(s(n)\)-space-bounded?	When \(M\) is total and for almost all inputs \(w\), <b>all computations</b> of \(M\) visit at most \(s(|w|)\) cells on each work tape. (Every branch must respect the bound, not just accepting ones.)	space::nondeterministic
Define NLOG and NPSPACE.	\(\text{NLOG} = \text{NSPACE}(\log)\) and \(\text{NPSPACE} = \text{NSPACE}(\text{poly})\).	space::nondeterministic
Why is DirPath (directed graph reachability) in NLOG?	On input \(\langle A, s, t \rangle\) for an \(n\)-node graph: store the current node (\(O(\log n)\) space) and a step counter (\(O(\log n)\) space). Nondeterministically guess the next node at each step. Accept if \(t\) is reached within \(n\) steps; reject otherwise. Total space: \(O(\log n)\).	space::nondeterministic
State the configuration-counting lemma for space-bounded TMs.	For an \(s(n)\)-space-bounded TM \(M\), there exists a constant \(d\) (depending only on \(M\)) such that for all \(n\):<br>(i) The number of distinct configurations on input length \(n\) is at most \(2^{d \cdot s(n)}\).<br>(ii) The depth of the computation tree is less than \(2^{d \cdot s(n)}\).	space::config_bound
Sketch the proof that the number of configurations is at most \(2^{d \cdot s(n)}\).	A configuration consists of: state (\(|Q|\) choices), input head position (\(n+2\) choices), and for each of \(k\) work tapes: contents (\(\leq |\Gamma|^{s(n)}\) possibilities) and head position (\(s(n)+1\) choices). The product is bounded by \(|Q|(n+2)(|\Gamma|^{s(n)}(s(n)+1))^k\), which for \(s(n) \geq \log n\) is at most \(2^{d \cdot s(n)}\).	space::config_bound
Prove that the computation tree depth is less than \(2^{d \cdot s(n)}\).	Let \(\ell = 2^{d \cdot s(n)}\). Suppose a branch has length \(\geq \ell\). Then there are \(\geq \ell + 1\) configurations along this branch but only \(\leq \ell\) distinct ones. By the pigeonhole principle, some configuration repeats, creating a loop — an infinite computation. This contradicts totality. So all branches have length \(< \ell\).	space::config_bound
What are the immediate inclusions between time and space?	\(\text{DTIME}(t(n)) \subseteq \text{DSPACE}(t(n))\) and \(\text{NTIME}(t(n)) \subseteq \text{NSPACE}(t(n))\). A \(t(n)\)-time-bounded TM can access at most \(t(n)+1\) cells on each work tape.	space::time_space_relations
How does \(\text{DSPACE}(s(n))\) relate to \(\text{DTIME}\)?	\(\text{DSPACE}(s(n)) \subseteq \text{DTIME}(2^{O(s(n))})\). A space-bounded machine has at most \(2^{O(s(n))}\) configurations, so it must halt within that many steps (or loop).	space::time_space_relations
State the full inclusion chain of major complexity classes.	\(\text{LOG} \subseteq \text{NLOG} \subseteq \text{P} \subseteq \text{NP} \subseteq \text{PSPACE} \subseteq \text{NPSPACE} \subseteq \text{EXP} \subseteq \text{NEXP}\)	space::time_space_relations
Which inclusions in the chain LOG ⊆ ... ⊆ NEXP are known to be strict?	\(\text{LOG} \neq \text{PSPACE}\), \(\text{NLOG} \neq \text{NPSPACE}\), \(\text{P} \neq \text{EXP}\), \(\text{NP} \neq \text{NEXP}\). Also \(\text{PSPACE} = \text{NPSPACE}\) (Savitch). It is unknown whether P ≠ NP, NP ≠ PSPACE, etc.	space::time_space_relations
State and prove: \(\text{NTIME}(t(n)) \subseteq \text{DSPACE}(t(n))\).	Given an NTM \(M\) running in time \(t(n)\), construct a DTM \(D\) that performs a depth-first search over \(M\)'s computation tree. \(D\) stores the current coding word (path from root) on an index tape and the current simulated configuration — both bounded by \(O(t(n))\). \(D\) backtracks when reaching halting or dead-end configurations. Since the tree has depth \(\leq t(n)\), total space is \(O(t(n))\).	space::NTIME_DSPACE
What is the key idea in proving \(\text{NSPACE}(s(n)) \subseteq \text{DTIME}(2^{O(s(n))})\)?	Instead of exploring the computation <b>tree</b> (which can be doubly exponential in size), construct the <b>configuration graph</b> (at most \(2^{O(s(n))}\) nodes). Use a BFS-like reachability algorithm: list all configurations, mark the initial one as reached, expand reached configurations by computing successors, and accept if any accepting configuration becomes reached. Total time: \(2^{O(s(n))}\).	space::NSPACE_DTIME
What is a space-constructible function?	A space bound \(s\) such that there exists an \(s(n)\)-space-bounded TM that computes \(1^n \mapsto 1^{s(n)}\). Examples: \(\log n\), linear, polynomial functions are all space-constructible. If \(s(n)\) is space-constructible, so is \(2^{s(n)}\).	space::savitch
State Savitch's Theorem.	Let \(s\) be a space-constructible space bound. Then \(\text{NSPACE}(s(n)) \subseteq \text{DSPACE}(s^2(n))\).	space::savitch
What is the key technique in Savitch's Theorem proof?	A recursive <b>divide-and-conquer</b> algorithm. To check if \(K_1 \xrightarrow{\leq 2^i}_N K_2\), iterate over all possible midpoint configurations \(K_{\text{mid}}\) and recursively check \(K_1 \xrightarrow{\leq 2^{i-1}} K_{\text{mid}}\) and \(K_{\text{mid}} \xrightarrow{\leq 2^{i-1}} K_2\). Recursion depth is \(\ell(n) = d \cdot s(n)\), each level stores \(O(s(n))\) for configurations, total space: \(O(s(n) \cdot s(n)) = O(s^2(n))\).	space::savitch
Why does Savitch's Theorem imply PSPACE = NPSPACE?	\(\text{PSPACE} \subseteq \text{NPSPACE}\) trivially. For the reverse: if \(L \in \text{NPSPACE}\), then \(L \in \text{NSPACE}(p(n))\) for some polynomial \(p\). By Savitch, \(L \in \text{DSPACE}(p^2(n))\). Since \(p^2\) is still a polynomial, \(L \in \text{PSPACE}\).	space::savitch
What is PSPACE = NPSPACE an analog of in time complexity?	It answers the space analog of the P vs NP question: nondeterminism gives at most a <b>quadratic</b> blowup in space. For time, the analogous question (P vs NP) remains open.	space::savitch
Define: a language \(B\) is hard for a complexity class \(C\).	Every language in \(C\) is p-m-reducible (polynomial-time many-one reducible) to \(B\).	space::completeness
Define: a language is complete for a complexity class \(C\).	It is <b>hard</b> for \(C\) and also <b>belongs to</b> \(C\).	space::completeness
What is a quantified propositional formula?	Defined inductively: (1) <b>Base</b>: any variable \(X \in \text{Var}\). (2) <b>Inductive</b>: if \(\psi, \psi'\) are QP formulas and \(X\) is a variable, then \(\neg \psi\), \((\psi \land \psi')\), and \(\exists X \psi\) are QP formulas. \(\forall X \phi\) is shorthand for \(\neg(\exists X \neg \phi)\).	space::QBF
What is prenex normal form for a quantified propositional formula?	The shape \(Q_1 X_1 \cdots Q_m X_m \phi\) where the \(X_i\) are mutually distinct variables, each \(Q_i \in \{\exists, \forall\}\), and \(\phi\) is a quantifier-free formula in CNF. Any quantified propositional formula can be converted to prenex normal form in polynomial time.	space::QBF
Define the language QBF.	\(\text{QBF} = \{\psi : \psi \text{ is a true sentence in prenex normal form}\}\). A sentence has no free variables, so its truth value is determined.	space::QBF
State the QBF completeness theorem.	\(\text{QBF}\) is \(\text{PSPACE}\)-complete. It plays the same role for PSPACE that SAT plays for NP.	space::QBF
Prove: QBF \(\in\) PSPACE.	Build a DTM that evaluates \(\psi = Q_1 X_1 \cdots Q_m X_m \phi\) by DFS over all \(2^m\) assignments. Maintain the current assignment \(w_3\) (length \(m \leq n\)) and a pointer. For each quantifier, try both values, combine with OR (for \(\exists\)) or AND (for \(\forall\)). Recursion depth is \(m \leq n\), each level uses \(O(1)\) extra space, evaluating \(\phi\) takes \(O(n)\). Total: \(O(n)\) space = polynomial.	space::QBF
What is the key idea in proving QBF is PSPACE-hard?	Reduce any \(A \in \text{PSPACE}\) to QBF. Encode configurations of a \(p(n)\)-space-bounded TM as propositional variables. Define \(\text{Comp}_n^i(K, K')\) asserting a path of length \(\leq 2^i\) exists from \(K\) to \(K'\). Use the <b>universal quantifier trick</b>: \(\text{Comp}_n^{i+1}\) uses only <b>one</b> copy of \(\text{Comp}_n^i\) (via \(\exists \tilde{K} \forall K_1 \forall K_2\)), keeping formula size polynomial. The naive Savitch-style definition uses two copies, giving exponential size.	space::QBF
Why can't we use the naive Savitch-style \(\text{Comp}\) definition for the QBF reduction?	The naive definition \(\text{Comp}^{i+1}(K,K') \equiv \exists \tilde{K}(\text{Comp}^i(K,\tilde{K}) \land \text{Comp}^i(\tilde{K},K'))\) has <b>two</b> copies of \(\text{Comp}^i\), giving size recurrence \(|\text{Comp}^{i+1}| = 2|\text{Comp}^i| + O(1)\), hence \(|\text{Comp}^i| = O(2^i)\) — exponential. The trick uses \(\forall K_1 \forall K_2\) with an implication to achieve the same semantics with <b>one</b> copy, giving linear growth: \(|\text{Comp}^i| = \text{poly}(n) \cdot i\).	space::QBF
Explain the universal quantifier trick: how does \(\forall K_1 \forall K_2\) encode two calls with one copy?	Like encoding \(P(a) \land P(b)\) as \(\forall x((x=a \lor x=b) \to P(x))\). The formula says: for all \((K_1, K_2)\), if they equal either \((K, \tilde{K})\) or \((\tilde{K}, K')\), then \(\text{Comp}^i(K_1, K_2)\) must hold. Only two assignments satisfy the antecedent, forcing both \(\text{Comp}^i(K, \tilde{K})\) and \(\text{Comp}^i(\tilde{K}, K')\). All others make the implication trivially true.	space::QBF
What does it mean for a complexity class to be closed under complement?	If \(L\) is in the class, then \(\bar{L} = \{0,1\}^* \setminus L\) is also in the class.	space::complement
Are deterministic complexity classes closed under complement? What about nondeterministic ones?	<b>Deterministic</b> time and space classes are trivially closed under complement (flip accept/reject). <b>Nondeterministic time</b>: it's open whether NP = co-NP. <b>Nondeterministic space</b>: yes, by the Immerman-Szelepcsényi theorem.	space::complement
State the Immerman-Szelepcsényi theorem.	Let \(s\) be space-constructible. Then \(\text{NSPACE}(s(n))\) is closed under complement.	space::complement
What is the key proof technique for the Immerman-Szelepcsényi theorem?	<b>Inductive counting.</b> Nondeterministically compute the number \(k_t\) of configurations reachable within \(t\) steps, for \(t = 0, 1, \dots, 2^{\ell(n)}\). Use this count to verify membership/non-membership in the set of reachable configurations. The counter allows the NTM to detect when it has enumerated <b>all</b> reachable configurations, enabling it to reliably reject (complement).	space::complement
What corollary follows from Immerman-Szelepcsényi for context-sensitive languages?	The class of context-sensitive languages is closed under complement. (Over binary alphabet, context-sensitive languages = \(\text{NSPACE}(n)\).)	space::complement
Summarize the four key theorems about nondeterministic space.	(1) \(\text{NTIME}(t(n)) \subseteq \text{DSPACE}(t(n))\) — nondeterministic time can be simulated in deterministic space.<br>(2) \(\text{NSPACE}(s(n)) \subseteq \text{DTIME}(2^{O(s(n))})\) — nondeterministic space simulated in exponential deterministic time.<br>(3) <b>Savitch</b>: \(\text{NSPACE}(s(n)) \subseteq \text{DSPACE}(s^2(n))\) — only quadratic blowup in space.<br>(4) <b>Immerman-Szelepcsényi</b>: \(\text{NSPACE}(s(n))\) is closed under complement.	space::summary
